# v0.3-13: Mobile Responsiveness

**Type**: AFK
**Blocked by**: None — can start immediately

## What to build

Make `/showcase` usable on mobile phone screens. The current implementation targets desktop (16:9). Mobile support should not compromise the desktop experience.

### Scope

Target phone screens: 375px–428px width (iPhone SE to iPhone Pro Max).

### Key adaptations

1. **Slide canvas**: Maintain 16:9 aspect ratio but scale to fit viewport width. Letterbox (black bars) top and bottom if needed — do not crop the photo.
2. **Category bar**: By default hidden. Swipe from left edge to reveal (or tap a small hamburger indicator). Auto-hide after selection. Same `C` key equivalent via touch.
3. **Top menu**: Tap top area to reveal. Larger touch targets (≥44px) for all menu items. Auto-hide after selection or tap elsewhere.
4. **Photo navigation**: Swipe left → next photo, swipe right → previous photo. This matches PRD 14.7 gesture requirements.
5. **Category navigation**: Swipe up → next category, swipe down → previous category (same direction as wheel). OR: long-swipe vertically for category change.
6. **Time axis**: Condensed to minimal — show only the dot indicator, hide the full text labels if space is tight.
7. **Caption text**: Reduce font size proportionally. Use `clamp()` for responsive font sizing.
8. **Bottom bar**: Show only position indicator ("3/28"). Time and location become visible on tap/hold.

### CSS strategy

Use CSS media queries and container queries where possible. Avoid JavaScript-based layout calculations. The slide renderer's normalized coordinate system (0–1) already handles proportional sizing — the main work is responsive font sizes, touch targets, and menu adaptation.

### Non-goals

- Not building a PWA (installable web app) — that's a future issue
- Not optimizing for tablets (tablets get desktop layout)
- Not adding mobile-specific features beyond what desktop already has
- No native app-like navigation gestures (bottom tabs, etc.)

## Acceptance criteria

- [ ] Showcase renders correctly at 375px–428px width
- [ ] Slide canvas maintains 16:9 aspect ratio
- [ ] Photo is fully visible (not cropped by viewport)
- [ ] Category bar accessible via left-edge swipe or tap
- [ ] Top menu accessible via top-area tap
- [ ] Touch swipe left/right navigates photos
- [ ] No text overflow or overlap on small screens
- [ ] Desktop layout unchanged (no regressions at ≥1280px)
- [ ] Existing keyboard + mouse navigation continues to work on desktop
- [ ] New tests: responsive layout verification for mobile breakpoints
