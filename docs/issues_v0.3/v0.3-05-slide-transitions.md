# v0.3-05: Slide Transitions & Animations

**Type**: AFK
**Blocked by**: None — can start immediately

## What to build

Add smooth visual transitions when navigating between photos, making the `/showcase` experience feel polished and cinematic. Transitions must be subtle — they enhance the viewing experience without distracting from the photo.

### Transition types

1. **fade** (default): current photo fades out, next photo fades in. Duration: 400–800ms.
2. **slide-left**: photos slide horizontally. `→` key slides from right to left; `←` key slides from left to right. Duration: 500–900ms.
3. **zoom-fade**: slight zoom (1.0→1.05) combined with fade. Duration: 600–1000ms.

### Implementation

- Use Vue `<Transition>` or `<TransitionGroup>` with CSS transitions
- Transition type comes from slide design metadata or global preference
- Between photos in the same category: use slide or fade transition
- Between categories: always use fade (avoid disorienting directional slide)
- Respect `prefers-reduced-motion` — disable transitions when user prefers reduced motion

### Animation tokens

Add `--kf-motion-duration-enter` and `--kf-motion-easing` CSS variables to control transition feel. These are part of the existing CSS token system.

### Non-goals

- No complex page-turn or 3D animations
- No animation that delays photo visibility by more than 1000ms
- No per-element staggered animations (photo/caption/timeline animating separately)

## Acceptance criteria

- [ ] Fade transition between photos works smoothly
- [ ] Slide transition direction matches navigation direction
- [ ] Category switch always uses fade transition
- [ ] `prefers-reduced-motion` disables animations
- [ ] Transition duration configurable via CSS tokens
- [ ] Rapid navigation doesn't cause animation stacking or visual glitches
- [ ] No white flash between transitions
- [ ] Existing keyboard + mouse navigation continues to work
- [ ] New tests: transition classes applied, direction correct, reduced-motion respected
