# v0.3-14: Playwright E2E + v0.3 Acceptance

**Type**: AFK
**Blocked by**: v0.3-01 through v0.3-13 (this is the final integration/validation issue)

## What to build

Add browser-automated end-to-end tests using Playwright, replacing the current curl+grep acceptance script approach for UI testing. The acceptance script still covers API-level verification, but UI interactions are verified through real browser tests.

### Playwright test suite

1. **Login flow**: visit `/login`, enter credentials, verify redirect to `/showcase`
2. **Showcase rendering**: verify slide appears with photo, timeline, and bottom bar
3. **Keyboard navigation**: `ArrowRight` → next photo, `ArrowLeft` → previous photo, `ArrowDown` → next category, `ArrowUp` → previous category
4. **Mouse navigation**: left-click → previous photo, right-click → next photo (verify no context menu), wheel down/up → category switch
5. **Menu behavior**: mouse to top → menu appears, mouse away → menu hides, `M` key toggles menu
6. **Category bar**: mouse to left → category bar appears with active category highlighted, `C` key toggles
7. **Photo detail**: navigate to photo detail page, verify metadata displayed
8. **Upload flow**: upload a test image, verify it appears in processing, wait for ready
9. **Admin functions**: admin can see admin-only UI elements, regular user cannot
10. **Empty state**: navigate to empty category, verify empty state message appears

### v0.3 acceptance script

Create `scripts/v0.3-acceptance.sh` and `just accept-v0-3` recipe:
- Step 1: Verify infrastructure (PostgreSQL, Redis, MinIO running)
- Step 2: Create test users (admin + member), login, verify cookies
- Step 3: Upload test photos to all categories
- Step 4: Run worker to process all photos
- Step 5: Verify API: showcase returns ready photos with valid slide designs
- Step 6: Verify API: admin endpoints (jobs, categories, audit logs)
- Step 7: Verify API: new v0.3 fields present (fill model, shadow model, presets if AI enabled)
- Step 8: Run Playwright tests (headless)
- Step 9: Verify backup/restore with v0.3 data
- Step 10: Verify mobile viewport rendering (Playwright mobile emulation)

### Playwright configuration

- Run against the local dev server (`localhost:3000`)
- Headless by default (for CI/acceptance script)
- Screenshots on failure for debugging
- Tests in `frontend/tests/e2e/` directory
- Run via `just test-e2e`
