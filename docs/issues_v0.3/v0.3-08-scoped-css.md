# v0.3-08: Scoped CSS Support

**Type**: HITL — security review required before merge
**Blocked by**: v0.3-02 (needs CSS variable infrastructure from Fill/Shadow models)

## What to build

Extend the AI CSS system from CSS Variables only to also support restricted scoped CSS blocks. AI can generate scoped CSS rules for fine-grained visual control, but every rule must pass through selector and property whitelists before reaching the DOM.

### Scoped CSS block

AI outputs scoped CSS as a string inside the design JSON:
```json
"styleTokens": {
  "cssVariables": { ... },
  "scopedCss": ".kf-slide-caption { text-wrap: balance; } .kf-slide-photo { filter: brightness(1.02); }"
}
```

### Selector whitelist

Only these selectors are allowed:
- `.kf-slide`, `.kf-slide[data-template]`, `.kf-slide[data-category]`
- `.kf-layer`, `.kf-layer[data-layer-id]`
- `.kf-photo-layer`, `.kf-text-layer`, `.kf-shape-layer`, `.kf-mask-layer`, `.kf-timeline-layer`
- `.kf-caption`, `.kf-meta`, `.kf-photo-frame`, `.kf-caption-panel`

Forbidden selectors: `html`, `body`, `#app`, `*`, `script`, `iframe`, `input`, `button`, `a[href]`, any comma-combined selector containing a forbidden selector, any parent escape selector.

### Property whitelist

Allowed: `color`, `background*`, `opacity`, `box-shadow`, `text-shadow`, `filter`, `backdrop-filter`, `mix-blend-mode`, `border*`, `border-radius`, `letter-spacing`, `line-height`, `font-*`, `text-*`, `transition*`, `animation*`, `transform`, `transform-origin`, `clip-path`, `mask-image*`.

Forbidden: `position`, `top/right/bottom/left`, `width/height/min-*`, `z-index`, `display`, `grid*`, `flex*`, `justify-content`, `align-items`, `overflow`, `pointer-events`, `cursor`, `content`, `visibility`.

### Sanitization pipeline

Must run on both backend (before DB save) and frontend (before DOM injection):
1. Parse CSS string into rules
2. For each rule: validate selector against whitelist → discard if invalid
3. For each declaration: validate property against whitelist → discard if invalid
4. Check for `@import`, `url()`, `javascript:`, `expression()` → discard entire block
5. Re-serialize safe rules and inject into scoped `<style>` element

### Security constraints

- No external resource loading (`url(http...)`, `@import`, `@font-face`)
- No JavaScript execution (`javascript:`, `expression()`, `behavior`)
- No global page manipulation
- No hiding the menu bar or category nav
- No modifying the slide container's layout properties

## Acceptance criteria

- [ ] Backend CSS sanitizer: parses CSS, validates selectors, validates properties
- [ ] Frontend CSS sanitizer: same logic, runs before DOM injection
- [ ] Whitelisted selectors pass through, forbidden selectors stripped
- [ ] Whitelisted properties pass through, forbidden properties stripped
- [ ] `@import`, `url()`, `javascript:`, `expression()` blocks rejected entirely
- [ ] Sanitized scoped CSS injected into slide via `<style scoped>` without affecting other slides
- [ ] No layout-breaking CSS reaches the DOM
- [ ] Tests: valid scoped CSS renders, malicious CSS blocked, edge cases (empty CSS, only forbidden rules, mixed valid/invalid)
- [ ] Security review sign-off before merge
