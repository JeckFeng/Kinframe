# v0.3-03: Texture + Vignette Layers

**Type**: AFK
**Blocked by**: v0.3-02 (needs Fill model for fill/noise types)

## What to build

Add two new layer types to the slide renderer: `texture` (film grain / noise texture overlay) and `vignette` (edge darkening overlay). Both are full-screen atmospheric layers that enhance visual quality without controlling layout.

### TextureLayer

Renders a full-screen noise/grain overlay using CSS or SVG noise. Controlled by:
- `fill.type: "noise"` — uses the noise fill type from the Fill model
- `opacity` — grain intensity (0–1)
- `blendMode` — typically "overlay" or "multiply"

Max 2 texture layers per slide.

### VignetteLayer

Renders a full-screen radial gradient from transparent center to darkened edges. Controlled by:
- `fill.type: "radialGradient"` — inner transparent, outer dark
- `opacity` — vignette strength (0–1)
- `blendMode` — typically "multiply"

Max 1 vignette layer per slide. The vignette should not obscure the photo center more than 15% brightness reduction.

### Integration points
- New Vue components: `TextureLayer.vue`, `VignetteLayer.vue`
- TypeScript types: `TextureLayer`, `VignetteLayer` interfaces
- LayerRenderer dispatch for both types
- Validator rules: fill type enforcement, max count limits, opacity bounds
- `layer_primitives.json` entries for both types
- Backend fallback generator may include texture/vignette in designs

## Acceptance criteria

- [ ] TextureLayer renders noise/grain overlay with configurable opacity
- [ ] VignetteLayer renders edge darkening with configurable strength
- [ ] Both layers respect zIndex ordering
- [ ] Validator enforces max count (texture ≤2, vignette ≤1)
- [ ] Validator rejects invalid fill types for these layers
