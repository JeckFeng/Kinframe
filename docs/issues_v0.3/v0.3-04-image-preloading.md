# v0.3-04: Image Preloading

**Type**: AFK
**Blocked by**: None — can start immediately

## What to build

Implement photo preloading in `/showcase` so that navigating between photos feels instant. The goal is <300ms perceived transition time.

### Preloading strategy

- After current photo renders, preload the next and previous photos in the current category
- Preload preview images (not originals) for adjacent photos
- Maintain a preload window: current ± 1 photo by default
- When the user switches categories, cancel in-flight preloads for the old category and begin preloading the new category

### Implementation approach

- Use browser-native `<link rel="preload">` or `new Image()` preloading
- Preload presigned URLs (which expire — ensure URLs are fresh)
- Show a brief shimmer or skeleton while the next image loads if it isn't cached
- Track which photo IDs are already loaded to avoid redundant preloading

### User-visible behavior

- Arrow key or mouse click → near-instant photo switch when adjacent photo is preloaded
- If preload hasn't completed, show current photo until next is ready (no flash of blank)
- The current photo remains visible during loading — no white screen

## Acceptance criteria

- [ ] Preloading fetches preview images for adjacent photos in current category
- [ ] Photo switch occurs in <300ms for preloaded adjacent photos
- [ ] Rapid key presses don't trigger duplicate preloads or race conditions
- [ ] Category switch cancels old preloads and starts new ones
- [ ] Presigned URL expiry handled (re-fetch if URL expired)
- [ ] No visual flash of blank/white between photo switches
- [ ] Existing navigation (keyboard + mouse) continues to work
- [ ] New tests: preload behavior, category switch cancels, rapid navigation
