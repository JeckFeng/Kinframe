## What to build

Create `design_presets.json` — a file of reusable, high-quality design fragments that AI can reference by ID instead of generating from scratch. This improves design consistency and reduces AI output complexity.

### Preset categories

1. **palettes** (调色板): curated color palettes with names and moods. At least 12 palettes covering warm/cool/cinematic/soft/bright/moody tones.
2. **shadows** (阴影): pre-configured shadow objects. At least 6: soft, dramatic, glow, inner, classic frame, subtle lift.
3. **masks** (蒙版): common mask configurations. At least 6: left fade, right fade, bottom fade, center vignette, top-bottom letterbox, corner darken.
4. **light-orbs** (光斑): decorative light orb shapes. At least 6 warm/cool/golden variants.
5. **timelines** (时间轴): timeline style presets. At least 4: minimal line, soft glow, year marks, dotted.
6. **fonts** (字体): font combination presets. At least 4: serif elegant, sans refined, mixed editorial, warm rounded.

### Usage

AI can reference a preset by ID:
```json
{ "presetRef": "misty_mountain_blue" }
```

The frontend or backend expands the preset reference before rendering/saving. This keeps design JSON compact and ensures preset quality.

### Preset expansion

- Backend expands presets during slide design validation/save, so stored designs are always self-contained
- Frontend also expands presets as a fallback during rendering
- If a presetRef is not found, log a warning and skip the layer
