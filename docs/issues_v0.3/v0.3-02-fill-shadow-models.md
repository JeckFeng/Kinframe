## What to build

Replace the current simple `style.gradient`/`style.color` fields with a structured Fill model and Shadow model across all layer types. Create the canonical `layer_primitives.json` config file that defines every layer type, its fields, allowed values, and limits.

### Fill Model

Each layer's fill is expressed as a structured object rather than flat style props:

```json
"fill": {
  "type": "solid | linearGradient | radialGradient | imageBlur | noise",
  "color": "#RRGGBB",           // for solid
  "angle": 135,                  // for linearGradient (degrees)
  "center": { "x": 0.5, "y": 0.5 }, // for radialGradient
  "radius": 0.6,                 // for radialGradient
  "stops": [
    { "color": "#RRGGBB", "opacity": 1.0, "position": 0.0 },
    { "color": "#RRGGBB", "opacity": 0.0, "position": 1.0 }
  ]
}
```

Rules: at least 2 stops per gradient, max 5 stops, position in [0,1], opacity in [0,1].

### Shadow Model

```json
"shadow": {
  "enabled": true,
  "type": "soft | dramatic | glow | inner",
  "x": 0, "y": 32,
  "blur": 80,
  "spread": -12,
  "color": "#000000",
  "opacity": 0.38
}
```

### layer_primitives.json

Create the canonical config file defining all layer types, their allowed fields, shapes, fill types, blend modes, max counts per slide, and zIndex range recommendations. This file is used by:
- Frontend validator (type-safe layer creation)
- Backend validator (schema enforcement)
- AI prompt (capability description)

The file should define all current layer types (background, image, text, shape, mask, timeline) plus future types (texture, vignette, frame).
