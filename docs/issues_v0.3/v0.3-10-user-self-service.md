# v0.3-10: User Self-Service Photo Editing

**Type**: AFK
**Blocked by**: None — can start immediately

## What to build

Allow regular (non-admin) users to edit the `user_message` field on photos they uploaded. Currently only admins can modify photo fields. This gives family members the ability to add or update their own captions without admin intervention.

### Behavior

- On the photo detail page (`/photo/[id]`), if the current user is the uploader of that photo, show an "Edit message" button
- Clicking opens an inline text input pre-filled with the current `user_message`
- On save, the caption priority rules re-evaluate:
  - If admin has set `caption_source = admin`, the admin override is preserved — user's new message is stored but `final_caption` stays as admin's value until admin resets
  - If no admin override, `final_caption = user_message` (user's message takes priority over AI)
- Users can only edit `user_message` — not category, not location, not AI fields
- Users can only edit their own photos — not photos uploaded by others

### Admin override protection

This is the critical rule: if an admin has manually set `final_caption` (making `caption_source = admin`), a regular user updating their `user_message` must NOT clear the admin's override. The admin must explicitly "Reset Caption" for the user's message to take effect.

### API endpoints

- `PATCH /api/photos/{photo_id}/message` — regular user endpoint, accepts `{"user_message": "new message"}`

## Acceptance criteria

- [ ] Regular user can edit `user_message` on their own photos
- [ ] Regular user cannot edit photos uploaded by others (403)
- [ ] Updating `user_message` does not override admin-set `final_caption`
- [ ] When no admin override exists, updating `user_message` updates `final_caption`
- [ ] Frontend shows edit button only to the photo's uploader
- [ ] Edited message appears immediately without page refresh
- [ ] Admin can still use "Reset Caption" to switch back to user_message priority
- [ ] Audit log records user message edits
- [ ] New tests: user edits own photo, user blocked from others' photos, admin override preserved
