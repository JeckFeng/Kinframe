# v0.3-09: AI Prompt Enhancement & Design Quality Scoring

**Type**: AFK
**Blocked by**: v0.3-01 (needs new templates), v0.3-02 (needs Fill/Shadow models), v0.3-07 (needs design presets)

## What to build

Two complementary improvements to the AI slide design pipeline:

### Part A: Enhanced AI Prompts

Rebuild the DeepSeek system prompt to include full capability descriptions drawn from the actual config files:
- All available templates with their slots, params, preferred categories, and moods
- All layer types with their fields and allowed values (from `layer_primitives.json`)
- All CSS variables with their purpose and value constraints
- All design presets with their IDs and visual descriptions
- Caption policy rules (user message priority, AI caption toggle)
- Output constraints (valid JSON only, no explanatory text)

The prompt should be modular: built dynamically from config files rather than hardcoded, so it stays in sync as templates/layers/presets change.

### Part B: Design Quality Scoring

After AI generates a slide design and it passes structural validation, run additional quality checks:

1. **Text occlusion check**: verify caption text area doesn't overlap the main photo area's center 60% region (where the subject likely is)
2. **Contrast check**: verify text color has sufficient contrast against the background color at the text position (WCAG AA minimum: 4.5:1 for normal text)
3. **Mask intensity check**: verify no mask layer has opacity > 0.65
4. **Layer count check**: verify total layers ≤ template maxExtraLayers + base layers
5. **Gradient stop check**: verify no gradient has >5 stops or <2 stops

If quality score is below threshold:
- Auto-retry AI generation once with the quality failure details in the retry prompt
- If retry also fails quality checks, use the best-scoring attempt
- If both fail structural validation, fall back to deterministic design

## Acceptance criteria

- [ ] AI prompt dynamically built from actual config files (templates, layers, presets)
- [ ] Prompt includes caption policy rules (user message priority, AI toggle)
- [ ] Prompt enforces JSON-only output (no explanatory text)
- [ ] Quality scoring implemented: text occlusion, contrast, mask intensity, layer count, gradient stops
- [ ] Quality failure triggers one retry with failure context in prompt
- [ ] Retry failure → uses best attempt or falls back to deterministic design
- [ ] Quality score and check results recorded in job metadata or audit log
- [ ] AI disabled mode unaffected (still uses fallback designs)
- [ ] New tests: prompt includes correct template/layer/preset info, quality checks pass/fail correctly, retry behavior
