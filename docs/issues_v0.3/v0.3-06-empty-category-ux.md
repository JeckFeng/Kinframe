# v0.3-06: Empty Category State + Photo Position Indicator

**Type**: AFK
**Blocked by**: None — can start immediately

## What to build

Two UX improvements for `/showcase`:

### Empty Category State

When the user navigates to a category that has no photos, display a graceful empty state instead of a blank or broken slide. The empty state shows:
- Category name
- A short, warm prompt: "这一组影像还在等待第一张照片。"
- An upload button linking to the upload page
- Visual hint that the user can switch to other categories

The empty state should still respect the slide template structure (background, timeline area) but show the prompt message centered.

### Photo Position Indicator

In the bottom information bar of each slide, show the current photo's position within the category:
- Format: "第 3/28 张" or "3 / 28"
- Updates immediately on photo switch
- For the first photo: "第 1/28 张"
- For the last photo: "第 28/28 张"

The position indicator should be visually subtle — small font, muted color, part of the existing bottom info bar alongside time/location.

## Acceptance criteria

- [ ] Empty category shows category name + prompt message + upload button
- [ ] Empty state slide respects slide layout (not raw white page)
- [ ] Empty state does not show broken image placeholders
- [ ] Photo position indicator shows "第 N/M 张" in bottom bar
- [ ] Position updates correctly on photo switch and category switch
- [ ] Position indicator responds correctly to category with single photo ("第 1/1 张")
- [ ] Both features work with keyboard navigation and mouse navigation
- [ ] Existing 42 frontend tests still pass
- [ ] New tests: empty state rendering, position indicator display, edge cases (1 photo, 0 photos)
