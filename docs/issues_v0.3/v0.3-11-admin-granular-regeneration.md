# v0.3-11: Admin Granular Regeneration

**Type**: AFK
**Blocked by**: None — can start immediately

## What to build

Currently, the admin "Regenerate" action on a photo re-runs the entire AI pipeline. Add granular options so admins can regenerate specific aspects without redoing everything.

### Regeneration options

From the admin photo detail page, admins can trigger:

1. **Regenerate Caption Only** — re-runs only the AI caption generation. Does not touch slide design, category, or location.
2. **Regenerate Template Only** — re-runs only template selection. Keeps existing layers, CSS tokens, and caption. Useful when the current template doesn't suit the photo.
3. **Regenerate CSS Tokens Only** — re-runs only the style token generation (colors, shadows, filters). Keeps template, layers, and caption.
4. **Regenerate Full Design** — current behavior, re-runs everything: template + layers + CSS tokens + caption (if AI caption enabled).
5. **Reset to Deterministic Fallback** — discards AI design entirely, generates a clean fallback design. Useful when AI designs are consistently poor.

### Implementation

- Each option maps to a specific AI job type or job parameter
- Backend API: `POST /api/admin/photos/{photo_id}/regenerate` with body `{"scope": "caption | template | css_tokens | full | fallback"}`
- Each regeneration creates an audit log entry recording the scope and result
- If AI is disabled and scope requires AI (caption/template/css/full), return a clear error — don't silently do nothing
- Fallback regeneration always works regardless of AI status

### UI

- Replace the single "Regenerate" button with a dropdown or segmented button group
- Show a confirmation dialog before triggering regeneration
- Show regeneration progress (job status polling)
- Disable the button while a regeneration job is in progress

## Acceptance criteria

- [ ] Five regeneration scopes available: caption, template, css_tokens, full, fallback
