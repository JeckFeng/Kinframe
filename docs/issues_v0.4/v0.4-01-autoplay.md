# v0.4-01: Auto-play System

## Type

AFK — can be implemented and merged without human interaction.

## Blocked by

None — can start immediately.

## What to build

Add an auto-play mode to the `/showcase` page. When active, photos advance automatically within the current category at a configurable interval. The user can toggle play/pause with the Space key and change the interval from the top menu bar.

This is a complete vertical slice through keyboard handling, timer state management, and menu UI — all within the showcase page.

### User experience

1. User presses **Space** on the showcase page → auto-play starts. Photos advance automatically (default 5s interval).
2. User presses **Space** again → auto-play pauses. The current photo stays.
3. Any manual navigation (arrow keys, mouse click, scroll, touch swipe) **pauses** auto-play — manual control always takes precedence.
4. The **top menu bar** gains a new control group showing the current interval: three buttons labeled `3s` / `5s` / `8s`. The active one is highlighted.
5. A subtle **play state indicator** appears near the bottom of the slide area when auto-play is active — a small pulsing dot or a thin progress bar that fills over the interval duration. It auto-hides after 2 seconds of mouse inactivity.
6. When auto-play reaches the last photo in a category, it wraps to the first (same as manual navigation loop behavior).
7. Auto-play **does not** cross category boundaries — it loops within the current category.

### What NOT to build

- Do NOT add new slide transition animations. Reuse existing `next-photo` transition.
- Do NOT add animation performance optimization work (no frame rate monitoring, no will-change tweaking).
- Do NOT change the existing category-switching behavior.
- Do NOT add auto-play for category switching.

## Technical approach

### State management

Add these reactive state variables to `frontend/pages/showcase.vue`:

- `isAutoPlaying: Ref<boolean>` — whether auto-play is currently active
- `autoPlayInterval: Ref<number>` — current interval in milliseconds (default 5000)
- `autoPlayTimer: Ref<ReturnType<typeof setInterval> | null>` — the active timer handle

### Keyboard handling

In the existing `onKeydown` function (showcase.vue ~line 217), add a `Space` case:

```typescript
case ' ':
case 'Space':
  e.preventDefault()
  toggleAutoPlay()
  break
```

### Timer logic

```typescript
function toggleAutoPlay() {
  if (isAutoPlaying.value) {
    stopAutoPlay()
  } else {
    startAutoPlay()
  }
}

function startAutoPlay() {
  isAutoPlaying.value = true
  autoPlayTimer.value = setInterval(() => {
    nextPhoto()  // reuse existing function
  }, autoPlayInterval.value)
}

function stopAutoPlay() {
  isAutoPlaying.value = false
  if (autoPlayTimer.value) {
    clearInterval(autoPlayTimer.value)
    autoPlayTimer.value = null
  }
}
```

Any manual navigation function (`nextPhoto`, `previousPhoto`, `moveCategory`, wheel handler, touch handler, mouse click handler) should call `stopAutoPlay()` before executing.

### Interval configuration

The `autoPlayInterval` ref stores one of three values: `3000`, `5000`, or `8000`. When the interval changes while auto-play is active, restart the timer with the new interval.

### Top menu UI

Add to the existing top menu `<div>` in showcase.vue (around line 288-336) a new control group, positioned in the center area between Gallery and the right-side links:

```html
<!-- Auto-play controls — only visible on showcase -->
<div class="flex items-center gap-1">
  <button @click="toggleAutoPlay"
    class="h-9 w-9 flex items-center justify-center rounded-lg transition-colors hover:bg-white/10"
    :title="isAutoPlaying ? '暂停自动播放' : '开始自动播放'">
    <!-- Play icon when paused, Pause icon when playing -->
    <Play v-if="!isAutoPlaying" :size="18" />
    <Pause v-else :size="18" />
  </button>
  <div class="flex rounded-lg bg-white/5 p-0.5">
    <button v-for="s in [3, 5, 8]" :key="s"
      @click="setAutoPlayInterval(s * 1000)"
      :class="autoPlayInterval === s * 1000 ? 'bg-white/15 text-white' : 'text-white/60 hover:text-white/80'"
      class="px-2 py-1 text-xs rounded-md transition-colors">
      {{ s }}s
    </button>
  </div>
</div>
```

The play/pause button uses icons from `lucide-vue-next` (already a project dependency). Import `Play` and `Pause` at the top of the `<script setup>` block.

### Play state indicator

Add a small indicator at the bottom-center of the slide area (not in the bottom info bar, but above it, within the slide rendering area):

```html
<!-- Auto-play progress bar -->
<div v-if="isAutoPlaying"
  class="absolute bottom-24 left-1/2 -translate-x-1/2 w-32 h-0.5 bg-white/10 rounded-full overflow-hidden"
  :class="indicatorVisible ? 'opacity-100' : 'opacity-0'"
  style="transition: opacity 0.6s">
  <div class="h-full bg-white/40 rounded-full animate-pulse"
    style="animation-duration: v-bind(autoPlayInterval + 'ms')">
  </div>
</div>
```

The indicator visibility is managed by a `pointermove`/timer pattern — show on mouse move, hide after 2s of no mouse movement. This is a simple `indicatorVisible` ref with a debounced hide.
