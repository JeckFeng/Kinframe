# v0.4-03: Map Album Page

## Type

AFK — can be implemented and merged without human interaction.

## Blocked by

None — can start immediately. Develop against `v0.4-02`'s API contract (the response schema defined in that issue). Use hardcoded mock data until the real endpoint is available. The two issues can merge in either order.

## What to build

Add a new `/map` page that displays geocoded photos as markers on a map of China. Each marker shows a photo thumbnail and caption on click. Users reach this page from the showcase top menu or by pressing `M`.

This is a frontend-only slice: new page route → map component → marker rendering → navigation entry points → E2E tests.

### User experience

1. User clicks **"地图"** in the top menu bar (or presses `M` on showcase) → navigates to `/map`.
2. The page shows a **full-viewport map of China** with photo markers clustered at their GPS coordinates.
3. Each marker is a small photo **thumbnail circle** (not a generic pin icon) — making it visually obvious that these are family photos.
4. **Clicking a marker** opens a popup showing: the photo thumbnail (larger), `final_caption` text, location name (e.g. "成都市, 四川省"), and `taken_at` date.
5. Clicking **"查看照片"** in the popup navigates to `/photo/[id]`.
6. The page has a **category filter bar** at the top (horizontal pill buttons: 全部 / 生活 / 摄影 / 萌宠) to filter markers by category.
7. Map supports standard **pan and zoom** (scroll wheel zoom, drag to pan).
8. A **"返回放映"** button in the top-left corner returns to `/showcase`.

### What NOT to build

- Do NOT build marker clustering (aggregating nearby markers at low zoom levels). Show individual markers at all zoom levels — family photo collections are small.
- Do NOT build a satellite/terrain layer toggle. Use the default map style.
- Do NOT build location search or address autocomplete.
- Do NOT build photo count badges per location.

## Technical approach

### Map library choice

Use **Leaflet** (`leaflet`) with OpenStreetMap tile layer. Rationale:
- Zero API key requirement — works offline and in private deployments
- Lightweight (~40KB gzipped)
- The PRD specifies China map; OpenStreetMap has adequate China coverage at city/district level
- No vendor lock-in

Install: `pnpm add leaflet @types/leaflet`

For the tile layer, use a Chinese tile server for better label rendering in China:
```
https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png
```
or consider `https://webrd0{s}.is.autonavi.com/appmaptile?lang=zh_cn&size=1&scale=1&style=8&x={x}&y={y}&z={z}` (Gaode/AMap tiles) for Chinese-language labels.

### Map initialization

Initialize Leaflet in `onMounted`:

```typescript
import L from 'leaflet'
import 'leaflet/dist/leaflet.css'

onMounted(() => {
  mapInstance.value = L.map(mapContainer.value!, {
    center: [35.86, 104.19],  // China center
    zoom: 5,
    zoomControl: true,
    attributionControl: false,
  })

  L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
    maxZoom: 18,
  }).addTo(mapInstance.value)

  markersLayer.value = L.layerGroup().addTo(mapInstance.value)

  // Add markers once data loads
  addMarkers()
})
```

Clean up in `onUnmounted`: `mapInstance.value?.remove()`

### Marker rendering

Each marker is a circular photo thumbnail using Leaflet's `L.divIcon` with custom HTML:

```typescript
function addMarkers() {
  markersLayer.value?.clearLayers()
  for (const photo of photos.value) {
    const icon = L.divIcon({
      className: 'kf-map-marker',
      html: `
        <div class="w-10 h-10 rounded-full border-2 border-white/80 shadow-lg overflow-hidden"
             style="background-image: url(${photo.thumbnail_url}); background-size: cover; background-position: center">
        </div>
      `,
      iconSize: [40, 40],
      iconAnchor: [20, 20],
    })

    const marker = L.marker([photo.gps_lat, photo.gps_lng], { icon })
      .bindPopup(popupContent(photo), { maxWidth: 280, closeButton: true })
      .addTo(markersLayer.value!)
  }
}
```

### Popup content

```typescript
function popupContent(photo: MapPhotoItem): string {
  const location = [photo.location_city, photo.location_region]
    .filter(Boolean).join(', ') || photo.location_name || ''
  const date = photo.taken_at
    ? new Date(photo.taken_at).toLocaleDateString('zh-CN', { year: 'numeric', month: 'long', day: 'numeric' })
    : ''
  return `
    <div class="kf-map-popup">
      <img src="${photo.thumbnail_url}" class="w-full h-32 object-cover rounded-lg mb-2" />
      ${photo.final_caption ? `<p class="text-sm text-stone-800 mb-1">${escapeHtml(photo.final_caption)}</p>` : ''}
      ${location ? `<p class="text-xs text-stone-500 mb-1">📍 ${escapeHtml(location)}</p>` : ''}
      ${date ? `<p class="text-xs text-stone-400 mb-2">${date}</p>` : ''}
      <a href="/photo/${photo.photo_id}" class="text-xs text-amber-700 hover:text-amber-900 font-medium">查看照片 →</a>
    </div>
  `
}
```

Basic HTML escaping for safety:

```typescript
function escapeHtml(text: string): string {
  const div = document.createElement('div')
  div.textContent = text
  return div.innerHTML
}
```

### Category filter

Reuse the existing `usePhotoCategories` composable (already used in showcase.vue) to get the category list. The filter bar is a row of pill buttons:

```html
<div class="flex gap-2 overflow-x-auto">
  <button @click="filterBy(null)"
    :class="activeCategory === null ? 'bg-white text-stone-900' : 'bg-white/10 text-white/70'"
    class="px-3 py-1.5 rounded-full text-sm whitespace-nowrap transition-colors">
    全部
  </button>
  <button v-for="cat in categories" :key="cat.slug" @click="filterBy(cat.slug)"
    :class="activeCategory === cat.slug ? 'bg-white text-stone-900' : 'bg-white/10 text-white/70'"
    class="px-3 py-1.5 rounded-full text-sm whitespace-nowrap transition-colors">
    {{ cat.name }}
  </button>
</div>
```

When filter changes, re-fetch data and update markers.
