# v0.5-delete-06: Playwright E2E + v0.5 Delete Acceptance

**Type**: AFK  
**Blocked by**: v0.5-delete-01, v0.5-delete-02, v0.5-delete-03, v0.5-delete-04, v0.5-delete-05

## What to build

Add full regression coverage for the delete system so hide/unhide and permanent delete become acceptance-tested product behavior rather than ad hoc manual checks.

This slice should validate both user and admin paths end-to-end.

## End-to-end scenarios

### Scenario A — Regular user hide/unhide

1. Upload photo
2. Confirm it appears in `showcase`
3. Hide it from showcase
4. Confirm it disappears from `showcase`
5. Confirm it still exists in `gallery`
6. Unhide it
7. Confirm it returns to `showcase`

### Scenario B — Admin permanent delete

1. Upload photo
2. Confirm photo exists in admin list/detail
3. Trigger permanent delete with confirmation
4. Wait for purge job success
5. Confirm photo no longer appears in:
   - admin list
   - gallery
   - showcase
   - map, if it had geodata
6. Confirm stale thumbnail/detail fetches fail cleanly

### Scenario C — Failure visibility

1. Simulate purge failure
2. Confirm admin sees failed job state and error text
3. Confirm no false success appears in UI

## Acceptance criteria

- [ ] Playwright test covers regular user hide/unhide flow
- [ ] Playwright test covers admin permanent delete flow
- [ ] Acceptance script verifies showcase exclusion after hide
- [ ] Acceptance script verifies full disappearance after successful purge
- [ ] Acceptance script verifies failure visibility for failed purge jobs
- [ ] Existing v0.3/v0.4 acceptance scripts still pass
- [ ] Delete system ships with a dedicated v0.5 acceptance path
