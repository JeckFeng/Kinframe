# v0.5-delete-04: Admin Permanent Delete Controls

**Type**: AFK  
**Blocked by**: v0.5-delete-03

## What to build

Add the admin UI needed to safely trigger permanent deletion. This slice should make the destructive operation understandable and hard to trigger by accident.

Deliver one complete admin path:

1. Admin opens a photo in operations UI
2. Admin chooses permanent delete
3. System shows a strong confirmation step
4. Admin confirms
5. UI shows pending/succeeded/failed state from the purge job

## Behavior

- Add permanent delete entry points in:
  - `/admin/photos`
  - `/photo/{id}` admin detail
- Permanent delete must have:
  - confirmation dialog
  - explicit destructive wording
  - clear note that this removes storage objects and DB data
- While purge job is pending/running:
  - disable repeated destructive clicks
  - show job progress/status feedback
- On success:
  - item disappears from admin list/detail navigation path
- On failure:
  - admin sees the job error and can retry through existing admin job tooling

## Permission rules

- Only admins see this control
- Regular users must never see permanent delete controls

## Acceptance criteria

- [ ] Admin list page exposes permanent delete action
- [ ] Admin detail page exposes permanent delete action
- [ ] Permanent delete requires explicit confirmation before request submission
- [ ] UI reflects pending/running/succeeded/failed purge job states
- [ ] Regular users do not see permanent delete controls anywhere
- [ ] On successful purge, photo is removed from admin browsing surfaces
- [ ] On failed purge, admin can discover the failure reason
- [ ] Frontend tests cover destructive confirmation and admin-only visibility
