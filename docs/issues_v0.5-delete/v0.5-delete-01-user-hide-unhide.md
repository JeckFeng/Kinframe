# v0.5-delete-01: User Hide / Unhide From Showcase

**Type**: AFK  
**Blocked by**: None — can start immediately

## What to build

Allow regular users to hide their own photos from the `showcase` experience without deleting any underlying data. This is not permanent deletion. The photo stays in MinIO, stays in the database, stays in `gallery`, and remains available for later unhide.

This slice should deliver one complete user path:

1. User opens a photo they own
2. User toggles whether the photo appears in `showcase`
3. `showcase` updates accordingly
4. `gallery` and photo detail still show the photo
5. The operation is permission-checked and test-covered

## Behavior

- Reuse `include_in_showcase` as the canonical visibility flag
- On the photo detail page, if the current user is the photo owner, show:
  - `Hide from showcase` when `include_in_showcase = true`
  - `Show in showcase` when `include_in_showcase = false`
- A regular user may only change this flag on their own photos
- Admins may still use this path too, but the core requirement is owner self-service
- Hidden photos:
  - must disappear from `/showcase`
  - must remain visible in `/gallery`
  - must keep all AI/geocoding/design data intact

## API behavior

- Extend the existing user-edit path rather than inventing a separate delete endpoint
- `PATCH /api/photos/{photo_id}` should accept `include_in_showcase`
- Permission rule:
  - owner: allowed
  - admin: allowed
  - other member: forbidden

## Acceptance criteria

- [ ] Photo owner can hide their own photo from `showcase`
- [ ] Photo owner can unhide their own photo back into `showcase`
- [ ] Hidden photo no longer appears in `/api/showcase`
- [ ] Hidden photo still appears in `/api/photos` and `/gallery`
- [ ] Non-owner member receives 403 when attempting this change
- [ ] Existing MinIO objects and slide designs remain untouched
- [ ] Photo detail page reflects current hidden/visible state immediately
- [ ] Backend tests cover owner success, non-owner rejection, and showcase exclusion
- [ ] Existing backend/frontend tests still pass
